In 2009, Ricardo Cabello — known online as Mr. Doob — submitted a Chrome Experiment to Google that proved something nobody had properly demonstrated: a browser could run a real-time rigid-body physics simulation without plugins, servers, or any infrastructure whatsoever. Every element of the Google homepage — the logo, the search bar, the footer links — fell, bounced, and collided using the same physics engine powering Angry Birds.

Google featured it in the Chrome Blog in 2012. It went viral globally. Dozens of regional forks and variants appeared within months.

In 2025, the experiment still runs. It still breaks in predictable, fixable ways. And as of November 2025, its name collides with an entirely new Google product — the Antigravity AI IDE — causing widespread search confusion that sends developers to the wrong documentation.

**One disambiguation before anything else:** if you searched "Google Antigravity" looking for the AI coding IDE launched November 2025 — this is not that product. This article covers the 2009 browser physics experiment, its architecture, its failure modes, and how to build your own version.

## How It Works: The Six-Layer Architecture

Before debugging or building, understanding the complete pipeline is essential. Every failure mode described later traces back to one of these six layers.

:::COMPONENT:DOMPhysicsLayerDiagram:::

### Layer 1 — HTML Snapshot

Mirror sites (elgoog.im, mrdoob.com) serve a self-contained static HTML file resembling the Google homepage. This is architecturally necessary: live Google.com sends strict CSP headers that block any third-party script.

### Layer 2 — DOM Traversal and Position Capture

At initialization, the script captures each element's position using `getBoundingClientRect()`. All reads must be batched together to prevent premature reflows.

### Layer 3 — Physics World Initialization

A physics world is created using Box2DJS (original) or Matter.js (modern). Gravity direction distinguishes Anti-Gravity (`y: -1`), standard Gravity (`y: +1`), and Zero Gravity (`y: 0`).

### Layer 4 — DOM Detachment

Each element is set to `position: fixed` with `transform` for placement. From this point, the element's visual position is entirely controlled by the physics engine.

### Layer 5 — The Render Loop

`requestAnimationFrame` drives a loop at 60fps. Each frame: step physics forward → read new positions → write `transform` to DOM. Two operations per frame.

### Layer 6 — Input Handling

Mouse and pointer events are translated into physics forces via a "mouse joint" — a spring constraint between the body and cursor. Releasing the spring throws the body.

## Hacks: 11 Problems, Root Causes, and Exact Fixes

### Hack 1: Script Does Nothing — Page Loads Normally

**Root Cause:** JavaScript disabled, or a script-blocking extension is silently preventing execution.

**Fix — in order of likelihood:**

1. Test in incognito mode. If it works there, an extension is the culprit.
2. Verify JavaScript is enabled in browser settings.
3. Open DevTools (F12) → Console. If you see "Refused to execute script" — you're running against live Google.com. Use a mirror site.
4. Disable extensions one by one. Primary suspects: uBlock Origin (hard mode), NoScript, Privacy Badger, Ghostery.

### Hack 2: No Effect on Mobile — Broken or Garbled Layout

**Root Cause:** The original uses `mousedown`/`mousemove`/`mouseup` with zero touch fallbacks. Mobile viewport breaks the body placement math.

**Fix:** Use elgoog.im — rebuilt with Pointer Events API. For custom implementations:

```javascript
// Use Pointer Events — works for mouse, touch, and stylus:
element.addEventListener('pointerdown', handler);
element.addEventListener('pointermove', handler);
element.addEventListener('pointerup', handler);

// iOS Safari: add passive: true
element.addEventListener('pointermove', handler, { passive: true });
```

### Hack 3: Extreme Lag, Browser Freeze, Fan Spinning Up

**Root Cause:** Writing `style.left` and `style.top` at 60fps forces full layout recalculation every frame. This is **layout thrashing**.

```javascript
// CAUSES LAYOUT THRASHING — avoid:
elements.forEach(el => {
    el.style.left = body.position.x + 'px';  // triggers reflow
    el.style.top  = body.position.y + 'px';  // triggers reflow again
});

// CORRECT — GPU compositor thread, zero layout cost:
elements.forEach(el => {
    el.style.transform =
        `translate(${x}px, ${y}px) rotate(${angle}rad)`;
});
```

### Hack 4: Search Bar Produces No Results

**Root Cause:** Google permanently deprecated the Web Search API in 2014. Fix: intercept form submission and redirect directly.

```javascript
document.querySelector('form').addEventListener('submit', e => {
    e.preventDefault();
    const q = document.querySelector('input[name=q]').value;
    window.open('https://www.google.com/search?q=' + encodeURIComponent(q));
});
```

### Hack 5: Elements Start in Wrong Positions

**Root Cause:** Race condition — script runs before fonts/images finish loading, capturing stale positions.

```javascript
// Wrong — fires before fonts and images have loaded:
document.addEventListener('DOMContentLoaded', initPhysics);

// Correct — fires after full page render:
window.addEventListener('load', initPhysics);

// Or if fonts are the issue:
document.fonts.ready.then(() => initPhysics());
```

### Hack 6: CSP Blocks Script Execution

**Root Cause:** Live google.com CSP always blocks third-party scripts. This is not a bug — it's a security policy.

**Fix:** Use mirror sites (elgoog.im). For your own domain, set CSP headers that whitelist your physics script's origin.

### Hack 7: Elements Escape Through Walls After Resize

**Root Cause:** Boundary walls are created once at viewport dimensions at init time. On resize, the physics wall positions don't update.

```javascript
let walls = createBoundaryWalls();

window.addEventListener('resize', () => {
    // Must remove old walls or duplicates accumulate
    walls.forEach(wall => World.remove(engine.world, wall));
    walls = createBoundaryWalls();
    World.add(engine.world, walls);
});
```

### Hack 8: Elements Tunnel Through Walls at High Speed

**Root Cause:** Discrete physics tunneling — element moves farther than its width in a single timestep.

```javascript
// Matter.js: cap maximum velocity in beforeUpdate event
Events.on(engine, 'beforeUpdate', () => {
    const maxVel = 20;
    Composite.allBodies(engine.world).forEach(body => {
        if (body.speed > maxVel) {
            Body.setSpeed(body, maxVel);
        }
    });
});
```

### Hack 9: iframe Embedding — Mouse Coordinates Are Offset

**Root Cause:** `event.clientX/Y` are relative to the current document viewport, not the parent page.

```javascript
// In the parent page:
document.addEventListener('mousemove', (e) => {
    const iframe = document.querySelector('iframe#gravity-frame');
    const rect = iframe.getBoundingClientRect();
    iframe.contentWindow.postMessage({
        type: 'MOUSE_MOVE',
        x: e.clientX - rect.left,
        y: e.clientY - rect.top
    }, '*');
});

// Inside the iframe:
window.addEventListener('message', (e) => {
    if (e.data.type === 'MOUSE_MOVE') {
        mouse.position.x = e.data.x;
        mouse.position.y = e.data.y;
    }
});
```

### Hack 10: Browser Extension Interference — Partial Elements Missing

**Root Cause:** Ad blockers remove DOM elements before the physics script runs. Null references throw and abort initialization.

```javascript
const elements = [...document.querySelectorAll('.physics-target')]
    .filter(el => el !== null && el.offsetParent !== null);

elements.forEach(el => {
    try {
        const body = createPhysicsBody(el);
        World.add(engine.world, body);
    } catch (e) {
        console.warn('Skipped element:', el, e.message);
    }
});
```

### Hack 11: HTTPS Mixed-Content Warning Blocks Physics Library

**Root Cause:** Old forks reference Box2D.js via `http://` CDN URLs — silently blocked on HTTPS pages.

```html
<!-- Wrong: -->
<script src="http://cdnjs.cloudflare.com/ajax/libs/matter-js/0.19.0/matter.min.js">

<!-- Correct: -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/matter-js/0.19.0/matter.min.js"
    integrity="sha384-[HASH]"
    crossorigin="anonymous">
</script>
```

Best practice: self-host all physics libraries to eliminate CDN dependency entirely.

## The PHYSICS Method: Implementation Framework

A complete deployment framework for DOM physics on any webpage from zero to production-ready.

**PHYSICS: Prepare → Hook → Initialize → Simulate → Control → Secure**

:::COMPONENT:PHYSICSMethodFramework:::

### Phase 1: Prepare — The Critical Batching Pattern

```javascript
// WRONG — read-write interleaving causes layout thrashing:
elements.forEach(el => {
    const r = el.getBoundingClientRect(); // READ — forces layout flush
    el.style.position = 'fixed';          // WRITE — invalidates layout
    el.style.left = r.left + 'px';
});

// CORRECT — batch ALL reads first, then ALL writes:
const rects = elements.map(el => el.getBoundingClientRect()); // all reads

elements.forEach((el, i) => {
    const r = rects[i];
    el.style.position   = 'fixed';
    el.style.left       = '0';
    el.style.top        = '0';
    el.style.width      = r.width  + 'px';
    el.style.height     = r.height + 'px';
    el.style.transform  = `translate(${r.left}px, ${r.top}px)`;
    el.style.willChange = 'transform';
    el.style.margin     = '0';
    el.style.transition = 'none'; // remove CSS transitions — they conflict
});
```

### Phase 2: Hook — Boundary Walls + Resize Handler

```javascript
function createBoundaryWalls(world) {
    const W = window.innerWidth;
    const H = window.innerHeight;
    const T = 60; // wall thickness in pixels

    const walls = [
        Bodies.rectangle(W/2,   H+T/2, W+T*2, T, { isStatic: true }), // floor
        Bodies.rectangle(W/2,  -T/2,   W+T*2, T, { isStatic: true }), // ceiling
        Bodies.rectangle(-T/2,  H/2,   T, H+T*2, { isStatic: true }), // left
        Bodies.rectangle(W+T/2, H/2,   T, H+T*2, { isStatic: true }), // right
    ];

    World.add(world, walls);

    window.addEventListener('resize', () => {
        walls.forEach(w => World.remove(world, w));
        createBoundaryWalls(world);
    });
}
```

### Phase 4: Simulate — The Render Loop with Physics Offset Fix

```javascript
function startRenderLoop(engine, bodies) {
    let running = true;

    document.addEventListener('visibilitychange', () => {
        running = !document.hidden;
    });

    function loop() {
        if (running) {
            Engine.update(engine, 1000 / 60);

            bodies.forEach(body => {
                if (!body._el) return;
                const { x, y } = body.position;

                // Subtract half-dimensions: converts center-origin (physics)
                // to top-left-origin (CSS) — the critical correctness fix
                body._el.style.transform =
                    `translate(${x - body._halfW}px, ${y - body._halfH}px) ` +
                    `rotate(${body.angle}rad)`;
            });
        }
        requestAnimationFrame(loop);
    }

    requestAnimationFrame(loop);
}
```

### Phase 5: Control — Runtime Gravity Switching

```javascript
const gravityModes = {
    normal:      { x:  0, y:  1   },
    antigravity: { x:  0, y: -1   },
    zero:        { x:  0, y:  0   },
    left:        { x: -1, y:  0   },

function setGravity(mode) {
    const g = gravityModes[mode];
    engine.gravity.x = g.x;
    engine.gravity.y = g.y;
    // Takes effect on the next physics step — no reset needed

// Rotating gravity — tumbling wheel effect:
let angle = 0;
function rotatingGravityLoop() {
    angle += 0.01;
    engine.gravity.x = Math.sin(angle);
    engine.gravity.y = Math.cos(angle);
    requestAnimationFrame(rotatingGravityLoop);
}

// DeviceMotion tilt gravity (iOS 13+ requires permission):
window.addEventListener('devicemotion', (event) => {
    const acc = event.accelerationIncludingGravity;
    if (acc) {
        engine.gravity.x =  acc.x / 9.8;
        engine.gravity.y = -acc.y / 9.8;
    }
});
```

### Explosion / Radial Impulse (Double-Click)

```javascript
function explode(originX, originY, radius = 300, strength = 0.05) {
    Composite.allBodies(engine.world).forEach(body => {
        if (body.isStatic) return;
        const dx   = body.position.x - originX;
        const dy   = body.position.y - originY;
        const dist = Math.sqrt(dx * dx + dy * dy);
        if (dist > radius || dist === 0) return;
        const falloff = (1 - dist / radius) * strength;
        Body.applyForce(body, body.position, {
            x: (dx / dist) * falloff,
            y: (dy / dist) * falloff
        });
    });
}

document.addEventListener('dblclick', e => explode(e.clientX, e.clientY));
```

## Mistakes Most Developers Make

**Running physics against live google.com.** The Content Security Policy on live Google.com will always block the script. No amount of troubleshooting changes this — use mirror sites or your own domain.

**Initializing on `DOMContentLoaded` instead of `window.load`.** DOMContentLoaded fires before fonts and images load. getBoundingClientRect() captures wrong positions. Every element ends up offset.

**Using `top` and `left` for position updates instead of `transform`.** This is the single most common cause of browser freeze. Writing `style.top` and `style.left` at 60fps forces full layout recalculation on every frame. The performance difference is 5–10× in favor of `transform`.

**Not removing old boundary walls before adding new ones on resize.** Adding without removing creates invisible duplicate collision surfaces inside the viewport. Elements bounce off invisible walls.

**Missing the center-to-corner physics offset.** Physics engines track by center point. CSS positions from top-left. Without subtracting half-dimensions from physics x/y, every element appears shifted by exactly half its size.

**Skipping `prefers-reduced-motion` in production.** Users with vestibular disorders can experience genuine physical discomfort. This is not optional for production use.
